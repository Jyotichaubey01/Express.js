Express.js

Express.js is a fast, lightweight, and flexible Node.js framework used to build web applications and APIs. It provides routing, middleware, error handling, and template engine support, making backend development easier and more efficient.

Express.js is a fast, lightweight, and flexible Node.js framework used to build web applications and APIs. It provides routing, middleware support, HTTP utilities, template engines, and error handling, making backend development easier and more efficient. It is the most popular framework for building server-side applications in Node.js.
Before using Express, install Node.js, npm, and a code editor like VS Code. Express can be added to any Node.js project using: npm install express This installs Express and adds it to your project dependencies. After installation, you can import Express and start building web applications and APIs quickly.
The Hello World example is the first Express application that demonstrates the basic structure of an Express app. It involves importing the Express module, creating an application instance, defining a route using app.get(), and starting the server using app.listen(). When the user visits the root URL (/), the server sends the message "Hello World from Express!" to the browser. This example helps beginners understand how Express handles requests and responses.
Basic Routing: Express uses routes to handle different HTTP requests such as GET, POST, PUT, and DELETE. Route Parameters: Dynamic values in the URL (e.g., /users/:id) that are accessed using req.params. Query Parameters: Key-value pairs after ? in a URL (e.g., /search?q=node) that are accessed using req.query. Summary: Routing controls URL requests, route parameters capture URL values, and query parameters pass additional data to the server.

Middleware

Middleware is a function in Express.js that executes during the request-response cycle. It acts as a bridge between the incoming request and the final route handler. Middleware can access and modify the req and res objects, execute code, end the request-response cycle, or pass control to the next middleware using next().

Middleware functions are executed in the order they are defined, creating a pipeline through which requests travel before reaching the route handler. This makes middleware useful for tasks such as logging, authentication, error handling, and data processing. If next() is not called, the request stops and no further middleware is executed.

Writing a middleware function

Every middleware function takes three arguments: req, res, and next. Here's a simple one that logs every request:

function logger(req, res, next) {
    console.log(`${req.method} ${req.url}`);
    next(); // pass control to the next middleware
}

app.use(logger);

Chaining multiple middleware

You can stack as many middleware functions as you need. Each one runs in order, and each must call next() to hand off to the one after it:

app.use(logger);
app.use(express.json());
app.use((req, res, next) => {
    console.log('Third middleware reached');
    next();
});

app.get('/', (req, res) => {
    res.send('Hello World');
});

The order above matters: logger runs first, then JSON parsing, then the third function, and only after all three call next() does the request reach the route handler.

Types of middleware

  • Application-level — bound to the app with app.use() or app.get(), runs for matching requests.
  • Router-level — same idea but bound to an express.Router() instance instead of the whole app.
  • Built-in — ships with Express: express.json(), express.urlencoded(), express.static().
  • Third-party — installed via npm, e.g. morgan (logging), cors (cross-origin requests), helmet (security headers).
  • Error-handling — takes four arguments (err, req, res, next) and catches errors thrown anywhere earlier in the pipeline.
